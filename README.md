
## DND Campaign Manager（beta 0.1）

一个**可运行、可维护、可逐步演进**的 DND 跑团资料管理工具。
当前版本以 **稳定可用** 为第一目标，所有功能均基于本地文件系统实现。

---

## ✨ 当前版本特性（beta 0.1）

### 🎲 跑团管理

* 创建跑团
* 删除跑团
* 在多个跑团之间切换
* 每个跑团数据完全独立存储

---

### 📂 内容分类管理

每个跑团固定包含以下分类：

* **人物卡（characters）**
* **怪物卡（monsters）**
* **地图（maps）**
* **剧情（notes）**

分类通过顶部按钮切换，当前分类具有明显的按下状态提示。

---

### 📝 文本内容功能

适用于：人物卡 / 怪物卡 / 剧情

* 新建 `.txt` 文件（自动补全扩展名）
* 根据分类自动生成基础模板（人物卡 / 怪物卡）
* 右侧窗口直接预览文本内容
* 双击文件使用系统默认文本编辑器打开
* **删除文件**（仅从软件界面隐藏，不删除实际文件）

---

### 🗺 地图功能（简化但稳定）

* 导入地图图片文件（复制到跑团目录）
* 支持常见图片格式（由系统与 Pillow 支持）
* 右侧窗口显示缩略预览（保持比例）
* 双击图片使用系统默认图片查看器打开原图
* **删除地图**（仅从软件界面隐藏，不删除实际文件）

---

### 🖥 界面与交互

* 左侧：跑团列表
* 顶部：分类切换与操作按钮
* 中部：文件列表
* 右侧：内容预览（文本 / 图片）
* 所有操作均为同步本地操作，无后台进程
* **安全删除**：删除功能仅隐藏文件，不会真正删除磁盘文件

---

## 🧠 设计原则

* **先稳定，再扩展**
* **不使用数据库**
* **不引入复杂架构**
* **文件结构即数据结构**

本项目明确避免过度设计，适合个人长期维护和逐步重构。

---

## 🛠 技术栈

* Python 3
* Tkinter（标准库 GUI）
* Pillow（图片预览）
* 本地文件系统存储

---

## 📁 项目结构

```
dnd_manager/
├── main.py              # 主程序（beta 0.1）
└── data/
    └── campaigns/       # 跑团数据目录（首次运行自动创建）
        └── 跑团名/
            ├── characters/
            ├── monsters/
            ├── maps/
            └── notes/
```

## 🚀 运行方式

### 1️⃣ 安装依赖

```bash
pip install pillow
```

（Tkinter 为 Python 标准库）

### 2️⃣ 启动程序

```bash
python main.py
```
